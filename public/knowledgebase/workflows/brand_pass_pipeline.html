The deliverable is a bundle

Three parts in one _work/<slug>/ folder: the blog (mirrors blog-sources/<slug>.md) + linkedin-institutional.md (firm voice) + linkedin-personal.md (founder voice).

Rename-on-date (stage 7)

The moment the publish date is decided, the work folder gains a YYYY-MM-DD_ prefix. Website slug stays bare; paths.work_dir() resolves either form, so no flag changes.

Separation of concerns

Claude does the vision (locate the fake logo, classify bg, pin bboxes) and authoring; the Python toolkit does the pixels deterministically and self-scans for the stray-blue defect class.

Hard rule

Never touch the website before the human gate. Staging writes to the working tree only — release is Michele's explicit commit + merge. Future-dated posts render as stubs until their date (UTC).

  1. Create the work folder — one bare _work/<slug>/ per blog (the date prefix is added later, at the human gate). Scaffold the three bundle files.
  2. Generate in NotebookLM — build the notebook, generate the infographic (website hero) + slide deck (body visuals). Manual today; auto-pull is the deferred Phase 4.
  3. Download — drop infographic.png + deck.pdf into <work>/inputs/.
  4. Author job.yaml — Claude inspects each image (Read tool): classify bg light/dark, pin a tight fake-logo bbox, decide mark + placement. Pinned bboxes make reruns deterministic.
  5. Run the toolkit — brand_pass.py --slug executes the job into <work>/processed/ and writes cohesion-report.md.
  6. Stray-blue gate — the report FAILS if a recolored wordmark kept a bright-blue stroke (the known bug class). Fix the bbox/mode and rerun, capped at ~2 iterations.
  7. Harmonize banner — whenever a logo was harmonized, the report prints a GUIDELINE EXCEPTION banner; surface it to Michele, never let it pass silently.
  8. Author the bundle — write post.md (the staged body, mirrors blog-sources/<slug>.md) plus the two LinkedIn companions; post_by equals the blog date:.
  9. Banned-phrase scan — the cohesion report runs the Node scanner on post.md against do-not-do.md; fix any hits.
  10. Human gate — present processed folder + report + staging command, then STOP. Michele approves and decides the publish date (or requests changes → back to job.yaml).
  11. Rename-on-date — with the date now set, rename _work/<slug>/ → _work/YYYY-MM-DD_<slug>/. Folder move only; bare --slug unchanged. If the date later changes, rename again.
  12. Stage — stage_blog_post.py --date copies images to assets/img/blog/<slug>/, writes blog-sources/<slug>.md, registers the slug in _build_blog_html.js. Working tree only — no commit, no deploy.
  13. Preview & publish — review at /blog/<slug>/, then PR → merge → Netlify. Future-dated posts render as stubs until their date (UTC); unstub merge lands on/after the publish day.

Folders carry a YYYY-MM-DD_ prefix once a date is set. Blog date:, LinkedIn post_by:, and the folder prefix must all agree (same-day rule).
