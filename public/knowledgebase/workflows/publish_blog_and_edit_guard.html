Bundle per post

Blog source (canonical) · _work mirror · hero · deck slides · infographic · linkedin-institutional.md · linkedin-personal.md · podcast link

Hard gates

12-row gate table, every row PASS or agreed N/A. verify_publish.py runs twice: --stage pre-pr (before PR) and --stage post-merge (cache-busted live fetch).

Guard signatures

orchestrator-wakeup · summary-log-wakeup · brief-watchdog · prism-daily-orchestrator · prism-daily-summary-log. Fails open on scan error — never wedges a session.

Timing rules

Blog date: = LinkedIn post_by: = work-folder prefix (same-day rule). Unstub PR merges on/after publish date UTC — early merges silently restore the stub.

  1. Resolve inputs — bare website slug, publish date, work folder Development/prism-notebooklm/brandpass/_work/<YYYY-MM-DD_><slug>/, Notion ticket. Same-day rule enforced up front: blog date: = LinkedIn post_by: = folder prefix.
  2. Confirm the post — read prism_website_project/blog-sources/<slug>.md (canonical) and quote title/slug/date back. The _work/<slug>/blog.md mirror loses on divergence.
  3. Brand-pass hero — scripts/brand_pass.py with the bare slug, then visual check: full triangle+wordmark lockup top-right, correctly sized, navy recolor complete.
  4. Brand-pass every deck slide — no sampling. Lockup present and sized, no off-palette gold in new assets, no leftover NotebookLM branding.
  5. Brand-pass infographic — same bar as the hero. A skipped infographic is a FAIL even when hero and deck passed.
  6. Interleave the deck — one slide per section with descriptive alt text, hand-placed in the post body. Staging via stage_blog_post.py needs the body frontmatter-stripped (double-frontmatter bug).
  7. Podcast branch — paired episode gets embedded; no episode gets recorded explicitly in the gate table, never silently skipped.
  8. LinkedIn companions — institutional (firm voice, no first person) + personal (founder voice, first person), each with the blog URL in the copy and a promo first-comment draft.
  9. Pre-PR verifier — verify_publish.py --stage pre-pr must exit 0: brand-pass artifacts, deck embeds + alt text, LinkedIn checks, same-day rule, podcast mapping, and the slug present in the hardcoded POSTS array in _build_blog_html.js.
  10. Gated PR + UTC timing — future-dated posts merge as stubs; the unstub PR merges on/after the publish date (UTC). An early merge silently restores the stub — recover with an empty-commit PR on publish day.
  11. Post-merge verifier — verify_publish.py --stage post-merge does the cache-busted live fetch and stub check itself. Netlify's webhook can silently skip a merge; "PR merged" is not "deployed".
  12. Close-out — canonical closing comment on the Notion ticket first, then status change. The 12-row gate table is shown to Michele before the post is declared live.

Guard behavior

PreToolUse hook on Edit|Write in .claude/settings.json → .claude/hooks/block-orchestrator-edits.js. Only scans when the target path ends .cmd or contains "orchestrator". Denies with a BLOCKED message while a signature process runs; fails open on any scan error.

Why it exists

Editing a .cmd mid-run corrupts it at the byte offset cmd.exe is reading and has caused an orchestrator double-run. The guard makes the CLAUDE.md "never edit a running automation file" rule mechanical.
